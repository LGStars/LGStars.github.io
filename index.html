1 关于地址和指针

真考链接

指针用于存放其他数据的地址

当指针指向变量时，利用指针可以指向该变量

当指针指向数组时，利用指针可以访问数组元素

当指针指向数组函数时，此时指针变量中存放的是函数的入口地址，可以通过指针调用该函数

当指针指向结构时，可以用指针引用结构变量的成员

1.1 地址：相当于每个变量的房间号。

1.2 内存大小：

（1）int 2个字节

（2）char 1个字节

（3）float 4个字节

（4）double 8个字节

1.3 指针：

（1）在C语言中对变量进行存取操作，实际上也就是直接对变量的地址存储单元进行操作，这种方式叫做“直接存取”。

（2）间接存取：即将变量的地址存放在另一种类型的变量中，从而通过这种新的变量类型来得到变量的值。

（3）通过地址能找到所需的变量单元，可以说：地址“指向”该变量单元。

（4）在C语言中，将地址形象的称为“地址”，意思是通过它能找到以它为地址的的内存单元，这里包含一个方向指向的意思。

（5）一个变量的地址称为变量的“指针”。一个专门用来存放另一个变量的地址的变量（即指针），则称它为“指针变量”。

2 变量的指针和指向变量的指针变量

2.1 指针变量的定义和引用

（1）类型名 *指针变量名，...；例如：int *p;

（2）*（星号）只是一个类型说明符。

2.2 指针变量的引用

（1）指针变量中只能存放地址（指针），将一个整型变量（或者任何其他非地址类型的数据）赋给一个指针变量是不被允许的。

（2）int *p;
    p = 300;(不合法的赋值)

（3）给一个指针赋空值：q = NULL 或 q = '\0' 或 q = 0；

界面的搭建

面向对象语言中有一种说法叫做万物皆对象，就像Java语言中，所有的类都是Object的子类。

在Android中，所能看到的全部控件都是View和它的子类展示的。

1 属性分析

（1）android：gravity - 表示控件里面的元素对齐方式。

（2）padding和margin用法一样，只不过margin是元素外边距，padding是内边距（元素的内部）。

（3）scaleType是缩放的方式，默认就是fitCenter，表示图片会填充控件，不会让图片变形，并且居中显示。

（4）android：scaleType="centerCrop"，以填满整个imageview为目的，等比例放大。

（5）android：scaleType="fitXY"，图片填充控件，但是允许图片拉伸。

（6）android：scaleType="center"，居中显示，超过剪切。

（7）android：scaleType="matrix"，不改变原图大小，从imageview左上角开始绘制原图。

（8）android：scaleType="fitStart"把原图按比例放大（缩小）到imageview的高度，显示在imageview的上部分位置。

（9）android：scaleType="centerInside"以原图完全显示为目的，将图片内容完整居中显示，通过按比例缩小原图的size宽（高）等于或者小于imageview的高（宽）。

2 布局介绍

Android的图形用户界面是由多个View和ViewGroup构建出来的。ViewGroup也是view的子类，是一个特殊的View。并且ViewGroup的不可见的。

（1）LinearLayout - 线性布局

（2）RelativeLayout - 相对布局

（3）FrameLayout - 帧布局

（4）GridLayout - 网格布局

（5）CoordinatorLayout - 材料设计引入的（处理各个子view之间的相互依赖关系）

（6）ConstraintLayout - 约束布局

（7）TableLayout - 表格布局（不常见）

（8）AbsoluteLayout - 绝对布局（基本废弃）

3.3 内存泄漏

Toast.makeText(MainActivity.this,”Hello”,Toast.LENGTH_SHORT).show();

（1）--该回收的内存由于各种原因没有被回收，还驻留在内存中。

（2）原因：如果在Toast消失之前，Toast持有了当前的activity，而此时，用户点击了返回键，导致activity无法被GC销毁，这个activity就引起了内存泄漏。

（3）解决：所有和当前无关的activity的Context都可以传进去。例如：Toast.makeText(getApplicationContext(),"Hello",Toast.LENGTH_SHORT).show();

1 一维数组的定义与引用 - 考核概率80%

真考链接

其中一维数组的引用是考查的重点。

在操作题中，常在修改题中考察一维数组的定义

1.1 一维数组定义

1.1.1 一维数组是指数组中的每个元素只带有一个下标的数组。

1.1.2 执行完 long array[10]，语句之后，C编程将会在内存中连续开辟10个存储单元，每个存储单元的名称就是每个数组元素的名称，如array[0]...，可以直接用名字来引用各个存储单元。

1.2 一维数组的引用

1.2.1 引用形式：数组名[下标表达式]。

1.2.2 注意：

    （1）一个数组元素实质上是一个变量名，代表内存中的一个存储单元，一个数组占据的是一连串连续的存储单元。

    （2）数组下标可以是整型常量，也可以是整型表达式。

    （3）只能逐个引用数组元素而不能一次引用整个数组。

    （4）在C语言运行过程中，系统并不自动检查数组元素的下标是否越界，所以数组两端都可能因为越界而破坏其他存储单元中的数据，甚至破坏程序代码，因此，要保证数组下标不能越界。

2 二维数组的定义与引用 - 考核概率90%

2.1 二维数组的定义

2.1.1 在C语言中，二维数组元素排列的顺序是：按行存放，即在内存中先按顺序存放第一行的元素，再存放第二行的元素。也占据一块连续的内存单元。

    注意：在C语言中明确规定：在定义二维数组时，后一个下标值不能省略，否则无法判断某一行元素的具体个数。

2.2.2 二维数组的一般形式：

    类型说明符 数组名[常量表达式][常量表达式]

    例如： int c[3][4] (三行四列)

2.2 其元素的引用

2.2.1 二维数组的表达形式：

    数组名[下标表达式1][下标表达式2]

2.2.2 注意：

    （1）：数组的下标可以是整型表达式，如c[3-1][3*2-2]。

    （2）：数组的元素可以出现在表达式中，也可以被赋值，如：a[1][3]=b[3][2]/2。

2.3 二维数组的初始化

2.3.1 在定义二维数组的同时可以给二维数组的各个元素赋初值。

    例如：float m[2][2] = {{1.5,3.2},{0.8}};

    全部的初值放在一对花括号中，每一行的初值又分别放在一对花括号中，之间用逗号隔开。当某行一对花括号内的初值个数少于该行的元素的个数时，系统会自动的给后面元素赋初值0.同样不能跳过前面的元素给后面的元素赋值。

3 字符数组 - 考核概率20%

真考连接

选择题中主要以读程序题要求考生写出程序运行结果的形式出现，是必考内容.

在操作题中，使用数组处理字符是常考内容。

3.1 字符数组的定义

3.1.1 字符数组就是数组中的每个元素都是字符，定义方法是逐个对数组的元素进行赋值。

3.1.2 如果花括号中提供的初值个数大于数组长度，则编译时会按语法错误处理。如果小于数组长度，则将剩下的元素自动定为空字符（'\0'）。

3.2 字符串和字符串结束的标志

3.2.1 C语言中，为了测定字符串的实际长度，C语言规定了一个字符串结束标识，以字符'\0'代表。

3.2.2 系统对字符串常量也自动加一个'\0'作为结束标志。例如：

    char c[] = "c program";

    该字符中共有9个字符，但在内存中占用10个字符，最后一个字节是'\0'是由系统自动加上的。

3.2.3 注意：'\0'代表ASCII码为0的字符，是一个“空操作符”，它什么也不干。在输出时也不输出'\0'，它只是一个结束的标识。

3.3 字符数组的输入/输出

3.3.1 方法：

    （1）用“%c”格式符将字符逐个输入或者输出。

    （2）用“%s”格式符将整个字符串一次输入或者输出。

3.3.2 注意：

    （1）用格式符“%s”在输出字符串时，printf（）函数输出项应该是数组名，而不是数组中的某个元素，如 “printf（“%s”，c[0]）;”是不对的。

    （2）在使用scanf（）函数时，输入项是字符数组名，不用再加取地址符“&”。

3.4 字符串处理函数

在使用这些函数时，必须在前面的命令行包括标准头文件“string.h”。

3.4.1 函数名称 - 调用形式 - 作用 - 说明

    （1）puts() - puts(字符数组) - 以'\0'结束 - 输出的字符串中可以包含转义字符

    （2）gets() - gets(字符数组) - - 只能一次输入一个字符串

    （3）strcpy() - strcpy(字符数组1，字符数组2) - 把字符数组2中的内容复制到字符数组1所指的存储空间中。函数返回数组1的值。 - 字符数组1空间 > 字符数组2的空间

    （4）strcat() - strcat(字符数组1，字符数组2) - 该函数将字符数组2所指的字符串的内容连接到字符数组1所指的字符串后面，并自动覆盖字符数组1的末尾'\0'。该函数返回字符数组1的地址值。 - 字符数组1空间应该足够大。

    （5）strcmp() - strcmp(字符数组1，字符数组2) - 该函数用来比较字符数组1和字符数组2所指的字符串的大小。若字符数组1 > 字符数组2，函数值 > 0（正数）；若字符数组1 = 字符数组2，函数值= 0；若字符数组1 < 字符数组2，函数值 < 0（负数） - 根据字符的ASCII依次对字符数组1和字符数组2所指的字符串进行比较，当出现第一对不同的字符时，既由这两个字符决定所在串的大小

    （6）strlen() - strlen(字符数组) - - 这一长度不包括串尾的结束标识'\0'。

2 安装各种需要的框架

目前流行的博客框架是Hexo,所以我们下面开始安装Hexo框架，所需要的工具主要有Git,Nodejs,Hexo。

2.1 安装Git

2.1.1 在安装Git的时候，我直接去官网下载的，但是官网下载速度实在是不敢恭维（37MB大小，差不多要半个小时，不知道是不是我网络的原因，但是下载其他软件挺快的），当然了，你可以直接去官网下载（https://git-scm.com/download），我是直接百度下载的，实测可以使用。

2.1.3 安装成功之后要进行测试，在程序中打开Git CMD，在CMD中输入git，如果出现了git的相关提示，则说明安装成功。

2.2 安装Nodejs

Node是一个服务器端 JavaScript 解释器，它可以帮助程序员构建高度可伸缩的应用程序，编写能够处理数万条同时连接到一个（只有一个）物理机的连接代码。下面就来介绍如何在Windows平台上下载、安装nodejs。

2.2.1 直接去官网下载nodejs安装包，然后直接点next即可。

2.2.2 安装完之后测试。

    2.2.2.1 首先按住Win+R,调出运行窗口，然后输入cmd，调出命令提示符窗口，在窗口中输入path,查看node及npm的环境变量是否配置好，如果出现了路径，则说明环境变量已经配置成功了。

    2.2.2.2 其次在命令提示符窗口中输入node -v，则会出来nodejs的版本号。

    2.2.2.3 然后再输入npm -v,如果出来版本号则说明成功安装。

2.2.3 新建一个文本文档，在里面输入以下代码。

    var http = require('http');
    server = http.createServer(function (req, res) {
    res.writeHeader(200, {"Content-Type": "text/plain"});
    res.end("Hello World\n");});
    server.listen(8000);console.log("httpd start @8000");

2.2.4 然后重新命名为test.js文件，将文件放在D盘根路径下（直接拖进去就可以啦，测试完可以删掉）。

2.2.5 在命令提示符窗口中，输入D：进入D盘（也就是存放test.js文件的目录），然后输入命令node test.js，可以看到输出一行结果httpd starts @8000。

2.3.2 当进度走完之后必须安装的软件已经全部安装成功，下面就可以生成博客，上传到Github中。

    git config --global user.name "你的名字"
    git commit 
    当运行完之后会弹出一个登陆邮箱，输入之后就可以正确发布了。

3.5.2 输入代码：hexo clean && hexo g && hexo d，然后执行完之后就可以在浏览器中访问你的博客了，网址：http//username.github.io.

4 编写自己的博客

4.1 当你再一次写完博文时，发布到github中。

4.2 在你博客站点文件夹下右键空白处，选择Git Bash Here，输入：hexo g，回车，生成静态页面。

4.3 再输入：hexo s，回车，然后到localhost:4000预览博客效果。

4.4 最后输入：hexo d，回车，同步到github上去就可以查看了。
