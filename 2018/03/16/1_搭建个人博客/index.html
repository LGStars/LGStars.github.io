搭建自己的博客 -徐军波

发表于 2018-03-16 | 热度: °C
字数统计: | 阅读时长 ≈

本文主要针对Windows用户。

搭建自己的博客主要原因：

1、搭建自己的博客可以自定义各种好看的界面。

2、没有各种广告。

3、记录自己的成长过程。

4、不受任何第三方的管理…

5、我就不多说什么了，安装之后可以亲自体会，下面进行安装，Let’s，go。

接下来我们开始创建

流程

1 创建Github域名和申请空间

1.1 注册

1.1.1 首先你需要注册一个Github账号（如果你已经有账号了，请直接看 1.2 创建仓库），    在注册过程中请注意username,这可能会影响到你的域名，以后申请成功后域名将会是username.github.io.

1.1.2 输入正确的邮箱之后Github会给你发一封邮件确认。

1.2 创建仓库

1.2.1 当你成功登陆Github后，在头像的头边会有一个加号，请点击加号创建一个仓库（New Respository）,Respository中的name一定要与前面的保持一致（username.github.io）,然后直接点击Create reponsitory

2 安装各种需要的框架

目前流行的博客框架是Hexo,所以我们下面开始安装Hexo框架，所需要的工具主要有Git,Nodejs,Hexo。

2.1 安装Git

2.1.1 在安装Git的时候，我直接去官网下载的，但是官网下载速度实在是不敢恭维（37MB大小，差不多要半个小时，不知道是不是我网络的原因，但是下载其他软件挺快的），当然了，你可以直接去官网下载（https://git-scm.com/download），我是直接百度下载的，实测可以使用。

2.1.2 在安装界面，选好文件夹位置之后，一直点NEXT，但是在之后，要选择“Use Windows’ default console window”,然后点击“Next”，直到finish即可。

2.1.3 安装成功之后要进行测试，在程序中打开Git CMD，在CMD中输入git，如果出现了git的相关提示，则说明安装成功。

2.2 安装Nodejs

Node是一个服务器端 JavaScript 解释器，它可以帮助程序员构建高度可伸缩的应用程序，编写能够处理数万条同时连接到一个（只有一个）物理机的连接代码。下面就来介绍如何在Windows平台上下载、安装nodejs。

2.2.1 直接去官网下载nodejs安装包，然后直接点next即可。

2.2.2 安装完之后测试。

    2.2.2.1 首先按住Win+R,调出运行窗口，然后输入cmd，调出命令提示符窗口，在窗口中输入path,查看node及npm的环境变量是否配置好，如果出现了路径，则说明环境变量已经配置成功了。

    2.2.2.2 其次在命令提示符窗口中输入node -v，则会出来nodejs的版本号。

    2.2.2.3 然后再输入npm -v,如果出来版本号则说明成功安装。

2.2.3 新建一个文本文档，在里面输入以下代码。

    var http = require('http');
    server = http.createServer(function (req, res) {
    res.writeHeader(200, {"Content-Type": "text/plain"});
    res.end("Hello World\n");});
    server.listen(8000);console.log("httpd start @8000");

2.2.4 然后重新命名为test.js文件，将文件放在D盘根路径下（直接拖进去就可以啦，测试完可以删掉）。

2.2.5 在命令提示符窗口中，输入D：进入D盘（也就是存放test.js文件的目录），然后输入命令node test.js，可以看到输出一行结果httpd starts @8000。 

2.2.6 然后在浏览器中输入网址127.0.0.1:8000可以看到网页输出Hello World,则表明测试成功。

2.3 安装Hexo

当你全部把以上软件安装成功之后，你已经完成70%了，马上就能看到自己的成果了。

2.3.1 打开Git Bash,在窗口中输入 npm install -g hexo，等待进度完成。

2.3.2 当进度走完之后必须安装的软件已经全部安装成功，下面就可以生成博客，上传到Github中。

3 第一次编写自己的博客，然后发布

接下来我们需要用Hexo初始化一个博客，然后更改一些配置，写自己的第一个博客，然后发布到自己的个人Github网站（你的网址：username.github.io）。

3.1 创建第一篇博客

3.1.1 打开Git Bash,然后再窗口中输入代码：hexo init username.github.io(可以将代码中的username换成自己的username，不换也无所谓，不影响)，然后看看路径，打开文件夹。

3.2 设置一些基础配置

3.2.1 主题安装（你可以去安装一些其他的主题），在窗口中输入代码：cd username.github.io(注意：username要保持一致)，然后再输入代码：git clone https://github.com/iissnan/hexo-theme-next themes/next。

3.2.2 Hexo中有许多主题可供你下载。

3.2.3 打开username.github.io/_config.yml修改几个键值对，一定要注意配置的键值对之间要有空格、要有空格、要有空格（重要的事情说三遍）。

    3.2.3.1 title： 博客名字(网站标题)

    3.2.3.2 author： 你的名字

    3.2.3.3 language： zh-Hans

    3.2.3.4 theme: next（主题名称）

    3.2.3.5 deploy:
            type: git(注意有的版本是github，可以多试试)
            repo： https://github.com/username/usernname.github.io.git

    3.2.3.6 subtitle: 网站副标题

    3.2.3.7 description: 网站描述

    3.2.3.8 timezone: 网站时区（默认使用电脑时区）

    3.2.3.9 网站存放在子目录中，例如http://yoursize.com/blog,则把url设置为http://yoursite.com/blog，并把root设为/blog/。(如果不是，则默认不用动，其他参数也是默认，你可以以后再改)。

3.3 开始写文章

3.3.1 在username.github.io/source/_posts下创建一个Test.md文件（当然名字可以随便取），然后用Markdown软件开始写（当然如果你没有这个软件，可以去网上下载，再查一下基本用法即可）

3.3.2 打开之后可以测试：“---（换行） title: Test(注意空格，在换行) ---（换行） > 下面可以写你的内容”

3.4 测试

3.4.1 输入代码：hexo s,执行完之后可以在浏览器中访问，地址：https://localhost:4000

3.4.2 然后安装自动部署发布工具hexo-deployer-git，输入代码：npm install hexo-deployer-git --save

3.5 发布

3.5.1 当没有错误时，在窗口中输入代码：在命令行中执行(因为是第一次运行)
    git config --global user.email "你的邮箱"
    git config --global user.name "你的名字"
    git commit 
    当运行完之后会弹出一个登陆邮箱，输入之后就可以正确发布了。

3.5.2 输入代码：hexo clean && hexo g && hexo d，然后执行完之后就可以在浏览器中访问你的博客了，网址：http//username.github.io.

4 编写自己的博客

4.1 当你再一次写完博文时，发布到github中。

4.2 在你博客站点文件夹下右键空白处，选择Git Bash Here，输入：hexo g，回车，生成静态页面。

4.3 再输入：hexo s，回车，然后到localhost:4000预览博客效果。

4.4 最后输入：hexo d，回车，同步到github上去就可以查看了。
